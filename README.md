# naaamgi.github.io

> 현직 모의해커가 운영하는 웹 보안 기술 블로그

## 소개

본 프로젝트는 현직 모의해킹(Penetration Testing) 전문가가 실무 경험을 바탕으로 운영하는 웹 보안 기술 블로그입니다. Jekyll 기반의 정적 사이트로 구축되었으며, 취약점 분석·모의해킹 기법·보안 도구 리뷰 등 100개 이상의 전문 보안 콘텐츠를 오픈소스 형태로 공개하고 있습니다.

단순한 기술 정리를 넘어, 현장에서 직접 경험한 공격 시나리오와 대응 방법을 구체적으로 기록함으로써 국내 보안 커뮤니티의 지식 접근성을 높이는 것을 목표로 합니다.

## 주요 콘텐츠

- 웹 취약점 분석 (Apache, SQL Injection, XSS 등)
- 모의해킹 기법 및 실습 가이드
- 보안 도구 리뷰 및 실전 테스트 결과
- AI 기반 자동화 보안 테스팅 도구(Strix) 실증 분석

## 담당 역할

본 블로그의 유일한 메인테이너로서 콘텐츠 기획·작성·배포 전 과정을 직접 담당합니다. Minimal Mistakes 테마를 기반으로 레이아웃과 기능을 직접 커스터마이징하였으며, Mermaid.js 다이어그램·utterances 댓글 시스템 등을 통합하여 기술 문서에 최적화된 환경을 구성했습니다.

현업에서 수행한 모의해킹 경험을 콘텐츠로 전환하여 보안 지식의 공유와 오픈소스 생태계 기여를 병행하고 있습니다.

## 이 프로젝트가 프로그램에 적합한 이유

현직 모의해킹 전문가가 직접 운영하는 보안 기술 블로그로, 실무에서 검증된 지식을 100개 이상의 포스트로 오픈소스화하여 국내 보안 커뮤니티에 기여하고 있습니다. 특히 AI 기반 보안 자동화 도구인 Strix를 실제 운영 저장소에 직접 적용하고 그 결과를 공개한 사례는, AI 보안 도구의 실효성을 현장 관점에서 검증한 국내 희소 레퍼런스입니다. 단순한 학습 기록이 아닌, 실전 공격 시나리오와 대응 방안을 구체적으로 기술함으로써 보안 정보 접근성을 높이고 오픈소스 보안 생태계 확산에 실질적으로 기여합니다.

## API 크레딧 사용 계획

API 크레딧을 활용하여 현직 모의해킹 업무와 연계한 보안 자동화 도구를 추가 개발하고 오픈소스로 공개할 계획입니다. 구체적으로는 신규 CVE 분석 및 영향도 평가 자동화, 취약점 리포트 초안 생성 파이프라인, 모의해킹 체크리스트 자동 생성 도구 등을 제작할 예정입니다. 이를 통해 보안 실무자의 반복 작업을 줄이고, 그 결과물을 블로그와 GitHub를 통해 커뮤니티에 환원하겠습니다.

## 추가로 알리고 싶은 내용

현업에서 모의해킹을 수행하면서, AI 도구가 보안 분야에서 실제로 어떤 수준의 취약점을 탐지하는지 직접 검증할 수 있는 위치에 있습니다. Strix를 운영 저장소에 직접 테스트하고 결과를 공개한 것도 그 연장선입니다. OpenAI Codex 지원을 통해 보안 자동화 도구 개발을 가속화하고, AI가 보안 실무에서 어디까지 신뢰 가능한지를 지속적으로 검증·공개하는 레퍼런스 역할을 이어가겠습니다.
